# 과제-05 주제 선정 가이드

이 문서는 `asmt-05.md`의 토의용 분류를 더 정교하게 보완하거나, 이후 다른 에이전트가 같은 작업을 이어받을 때 참고할 수 있도록 만든 공통 작업 메모다. 2026년 4월 20일 기준으로 정리했다.

## 무엇을 promising 하다고 볼 것인가

- 학생 제안들이 하나의 질문축으로 다시 모일 수 있고, 찬반 또는 경쟁 이론의 라인이 분명해야 한다.
- 학부 3-4학년이 감당할 수 있는 수준의 canonical 쟁점이 살아 있어야 한다.
- 최근에도 논의가 이어지고, 입문 수준에서 찾을 수 있는 대표 문헌과 반론 문헌이 모두 확보되어야 한다.
- 정책평가형 주제라면 단순 효과 비교가 아니라 규범적 판단 기준까지 연결될 수 있어야 한다.
- 기술이나 사례가 중심이더라도 결국은 “무엇이 정당한가” “어떤 기준이 우선하는가” 같은 논증형 질문으로 회수될 수 있어야 한다.

## 느슨한 패키지를 다루는 원칙

- 패키지 내부 응집도가 약하다고 해서 그 분류 전체를 곧바로 탈락시키지는 않는다.
- 먼저 그 분류 아래의 개별 학생 주제들이 독립된 논증형 에세이로 설 수 있는지 따로 본다.
- 이런 경우 코멘트 문서에는 패키지 제목을 유지하되, 짧은 `공통 메모` 아래에 학생별 가능성을 `높음`, `중상`, `중`처럼 따로 적는 편이 좋다.
- 따라서 실제 토의에서는 “패키지를 살릴 것인가”와 “그 안에서 어떤 개별 주제가 살아남는가”를 구분해 읽게 만든다.

## 피해야 할 신호

- 쟁점이 지나치게 오래되어 현재는 일부 연구자만 다루는 고전적 논쟁으로만 남아 있는 경우
- 최근 문헌은 많지만 대체로 실험 설계나 계량 검증이 핵심이라 문헌 중심 에세이로 소화하기 어려운 경우
- 학생들이 잡은 질문이 너무 넓어서 실제로는 선언적 찬반이나 허수아비 반박으로 흐르기 쉬운 경우
- 문헌이 있더라도 지나치게 전문화되어 입문 수준에서 canonical한 논쟁선을 정리하기 어려운 경우
- 한 분류 안의 학생 주제들이 서로 다른 질문을 하고 있어 패키지로 토론할수록 오히려 흐려지는 경우

## 문헌 검증 체크리스트

- 저자, 제목, 연도, 저널 또는 출판사 표기를 다시 확인한다.
- 가능하면 출판사, 학회, 저널, DOI, Cambridge Core, Oxford Academic, AEA, NBER 같은 1차 서지 페이지로 재확인한다.
- 학생 문서의 헤더와 실제 서지 정보가 어긋나는 경우, 본문을 바로 고치기보다 코멘트 문서에서 먼저 정제 메모를 남긴다.
- 입문용 대표 문헌만 둘 것이 아니라, 서로 경쟁하는 대표 문헌을 함께 적는다.
- 너무 많은 참고문헌을 나열하기보다 “이 축을 실제로 살릴 수 있는 최소 canon”을 먼저 제시한다.

## 반복 가능한 작업 절차

1. `asmt-05.md`의 현재 분류를 읽고, 학생별 문서가 실제로 같은 질문축에 들어가는지 다시 점검한다.
2. 각 분류마다 “살릴 이유”, “버릴 이유”, “남기려면 어떻게 좁혀야 하는가”를 한두 문단으로 쓴다.
3. 그 분류에서 실제로 경쟁 구도를 형성하는 대표 문헌 3-5개를 뽑고, 실재 여부와 기본 서지를 검증한다.
4. 학생 문서에 서지 오류가 보이면, 해당 분류 코멘트 아래에 정제 메모를 남긴다.
5. 메인 `asmt-05.md`는 요약과 토의 구조를 유지하고, 긴 검증 메모는 별도 코멘트 문서로 빼는 편이 화면상 읽기 좋다.
6. 패키지 응집도가 느슨한 분류는 “약하다”라고만 쓰지 말고, 개별 학생 주제별 생존 가능성을 따로 평가한다.

## 문서 배치 원칙

- 메인 페이지 `asmt-05.md`: 학생 요약, 토의용 분류, 요약 표
- 보강 코멘트 문서: promising 판단 참고, 경고 신호, 검증된 대표 문헌, 서지 정제 메모
- `README2.md`: 이런 문서가 왜 생겼는지와 다음 작업자가 어디서 이어받아야 하는지 기록

## 코멘트 문서 생성 규칙

- 파일명은 `asmt-05-commentary.md`로 통일한다.
- 문서 제목과 본문 최상단 제목은 모두 `과제-05 검토 메모`로 통일한다.
- permalink는 `/asmt-05/commentary/`를 유지한다.
- 코멘트 문서는 메인 `asmt-05.md`의 확장 문서로 보고, 메인 페이지에는 짧은 안내와 링크만 둔다.
- 긴 판정 메모, 대표 문헌, 서지 정제 메모, 느슨한 패키지의 개별 주제 평가는 코멘트 문서에 둔다.

## nav_order 운영 원칙

- `asmt-05` 하위의 학생 문서는 이미 학생 번호 기반 `nav_order`를 쓰고 있으므로, 나중에 검토 문서를 추가할 때 학생 파일을 다시 고치지 않는 편이 좋다.
- 이런 검토/코멘트 문서는 `nav_order: 0`으로 두면 학생 문서들보다 먼저 오고, 학생 문서의 기존 순서를 건드리지 않아도 된다.
- Just the Docs 공식 문서상 `nav_order`는 숫자 값을 허용하므로 `0`도 사용할 수 있다.
